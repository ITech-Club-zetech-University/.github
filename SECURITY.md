# Security Policy

## Supported Versions
Only active and maintained repositories are eligible for security updates.

## Reporting a Vulnerability
If you discover a security issue:

1. Do **not** open a public issue  
2.Contact us
3. Include:
   - Steps to reproduce  
   - Potential impact  
   - Proposed mitigation (optional)

We aim to acknowledge reports within **48 hours**.

## Responsible Disclosure
We request that you:
- Give us reasonable time to fix issues  
- Avoid exploiting vulnerabilities  
- Avoid sharing details publicly until patched

## Security Expectations
Contributors should:
- Never commit secrets or credentials  
- Use `.env` files and GitHub Secrets  
- Follow secure coding practices
